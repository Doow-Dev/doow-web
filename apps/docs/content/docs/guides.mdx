## Workflow guides

Guides explain how a team uses Doow repeatedly, not just how a button works.
Each guide must name the role, the workflow trigger, the expected outcome, and
the checks that prevent drift.

## Starter guide areas

- Subscription review cadence
- Application owner cleanup
- Integration readiness
- Renewal evidence gathering

<Callout title="Guide quality">
  A guide is complete when a new operator can follow it without needing product
  context from the person who wrote it.
</Callout>
