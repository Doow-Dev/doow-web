## What Doow needs

Doow operates on three classes of data: **transactions** (corporate cards,
expense reports, AP feeds), **identity** (who owns or uses what), and
**ledger** (so spend can be reconciled to a category and cost centre).

You can start with just transactions and identity. Ledger sync becomes
useful when you scale beyond a few teams.

## Supported sources

A short list of supported sources ships with the launch. Custom sources
can be wired through the import API once that lands.

## Test mode

Every connector supports test mode. We recommend doing the first review
against a sandboxed source before flipping production credentials.
