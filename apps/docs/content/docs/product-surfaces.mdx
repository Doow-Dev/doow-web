## The four surfaces

Every workspace exposes the same four surfaces: **Inbox** (incoming
review work), **Catalog** (the canonical list of apps), **Renewals**
(time-ordered upcoming renewals), and **Reports** (saved rollups).

A fifth surface, **Audit**, is admin-only and records every change to
ownership, approvals, and policies.

## Composition rules

Surfaces share a common shell — left rail, body, right rail — and a
common set of filter primitives. Anything you learn in one surface
applies to the others.

## Custom surfaces

Custom surfaces are not yet supported. The roadmap targets it for the
business plan once the catalog API stabilises.
