## Event shape

Every webhook delivers a JSON payload with three top-level keys: `id`
(unique event ID, idempotent), `type` (event name in dotted form), and
`data` (the resource that changed). Events are deduped on `id`.

## Supported events

The first set covers the highest-leverage cases: `transaction.created`,
`approval.completed`, `renewal.opened`, and `vendor.archived`. Additional
events ship as workflows mature.

## Delivery

Webhooks are delivered with at-least-once semantics. We retry up to six
times with exponential backoff over 24 hours. Endpoints should be
idempotent on the event `id`.

## Signatures

Each request is signed with HMAC-SHA256 using the workspace webhook
secret. Verify the `X-Doow-Signature` header before trusting the body.

<Callout title="Beta">
  Webhooks are in beta. The event list and payload shapes may change
  before general availability. Pin to the `2026-05` API version while
  testing.
</Callout>
