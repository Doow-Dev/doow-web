## Prepare your workspace

Doow docs start from the same production monorepo as the web app. Keep docs
content in `apps/docs/content/docs`, keep docs-specific rendering in
`apps/docs/src/components/docs`, and share only stable primitives from packages.

<Callout title="Production rule" tone="warning">
  Docs content and docs UI can ship independently from the landing page, but
  they must pass the same monorepo verification loop before deploy.
</Callout>

## Create the first page

<Steps>
  <ol>
    <li>Add an MDX file under `apps/docs/content/docs`.</li>
    <li>Set frontmatter with `title`, `description`, `slug`, `section`, and `order`.</li>
    <li>Add the slug to the docs navigation registry.</li>
    <li>Run `pnpm --filter @doow/docs check:content`.</li>
  </ol>
</Steps>

## Verify locally

<Tabs id="docs-verify" items="pnpm, npm, yarn">
  <CodeBlock language="bash" code="pnpm --filter @doow/docs check:content && pnpm --filter @doow/docs build" />
  <CodeBlock language="bash" code="npm run check:content --workspace @doow/docs && npm run build --workspace @doow/docs" />
  <CodeBlock language="bash" code="yarn workspace @doow/docs check:content && yarn workspace @doow/docs build" />
</Tabs>
