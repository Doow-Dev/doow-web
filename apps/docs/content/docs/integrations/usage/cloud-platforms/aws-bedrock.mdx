Connect AWS Bedrock to pull per-model token usage into Doow via CloudWatch metrics.

## What you need

- An AWS account with Bedrock enabled
- An **IAM user or role** with read-only access to CloudWatch and Bedrock usage metrics
- The AWS region (or regions) where your Bedrock workloads run

## Creating a read-only IAM credential

Doow needs an AWS access key pair (Access Key ID + Secret Access Key) belonging to an IAM user with the following minimum permissions:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "cloudwatch:GetMetricData",
        "cloudwatch:ListMetrics"
      ],
      "Resource": "*"
    }
  ]
}
```

**Steps to create the IAM user:**

1. Go to the [AWS IAM Console](https://console.aws.amazon.com/iam)
2. Navigate to **Users → Create user**
3. Name it something identifiable (for example, `doow-bedrock-readonly`)
4. Attach a policy with the permissions above (or create a custom policy)
5. Go to **Security credentials → Create access key**
6. Select **Third-party service** as the use case
7. Copy the **Access Key ID** and **Secret Access Key** — you will not see the secret again

## What Doow reads

| Metric | Description |
|---|---|
| `input_tokens` | Prompt tokens sent to each Bedrock model, per day |
| `output_tokens` | Completion tokens returned by each model, per day |
| Model | Which model was invoked (for example, `anthropic.claude-3-sonnet`, `amazon.titan-text`) |
| Region | The AWS region where the usage occurred |

Usage is pulled from CloudWatch and aggregated at the day level per model and region.

## What Doow does not read

- The content of any inference requests or responses
- Your S3 data, other AWS services, or account-level billing data beyond Bedrock metrics
- IAM credentials, secrets, or any security configuration

## How to connect

1. Go to **Company Settings → Integrations** in your Doow workspace
2. Find **AWS Bedrock** and click **Connect**
3. A dialog opens asking for your credentials
4. Enter your **Access Key ID** and **Secret Access Key**
5. Select the AWS region(s) where your Bedrock workloads run
6. Click **Connect**
7. Doow verifies the credentials and begins the initial usage sync

## Multi-region setups

If you run Bedrock workloads across multiple AWS regions, you can add each region when connecting. Doow will pull metrics from all selected regions and aggregate them in your catalog.

## Multi-account setups

If your organization uses multiple AWS accounts for Bedrock, add a separate connection for each account's IAM credential.

## Confirm the sync worked

After connecting, open the integration detail page from **Company Settings → Integrations**. A healthy cloud sync should show a connected state, a recent sync timestamp, and usage rows for the expected account, project, region, deployment, or model.

If the connection succeeds but no usage appears, verify that the credential has read access to the correct cloud account, project, region, monitoring data, and billing or usage export source.

## Troubleshooting

Use these checks when cloud usage data is missing or incomplete:

- Confirm the credential belongs to the intended cloud account, subscription, or project.
- Confirm it can read the required monitoring, billing, or usage export data.
- Confirm the expected regions, deployments, or models have recent usage.
- Check whether the cloud provider reports usage with a delay.
- Check the integration event log for permission or sync errors.

### AWS Bedrock-specific checks

| Symptom | Likely cause | Next action |
|---|---|---|
| No usage appears for a region | The IAM user cannot read CloudWatch metrics in that region | Confirm the policy covers each Bedrock region you expect Doow to read |
| Some models are missing | Bedrock usage exists in a different account or region | Add a connection for each AWS account and region that owns usage |
| Sync fails with access denied | The IAM policy is missing required read permissions | Re-check the read-only policy attached to the IAM user |

## Disconnecting

Go to **Company Settings → Integrations**, find AWS Bedrock, and click **Disconnect**. The stored credentials are deleted immediately. You can also delete the IAM user from the AWS Console to fully revoke access.
