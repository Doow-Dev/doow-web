The instrumentation SDK lets you push usage events from your own applications directly into Doow using the `@doow/track` SDK. Unlike direct provider API integrations (where Doow pulls from a provider's API), the SDK is a push route — your application emits events in real time as usage occurs.

## When to use this

Use the instrumentation SDK when:
- You are building a product that you want to meter and track against a contract in Doow
- You want usage data in Doow that is not available from a vendor's usage API
- You need attribution at a granularity (per customer, per feature, per environment) that vendor APIs don't expose

## Choose a method

Pick the method that matches where your application runs.

| Method | Use it when | Setup guide |
|---|---|---|
| Node.js SDK | You can add `@doow/track` directly to a Node.js application | [Node.js SDK](/integrations/usage/instrumentation-sdk/nodejs) |
| Sidecar | Your service is not written in Node.js, or you want telemetry outside the main process | [Sidecar](/integrations/usage/instrumentation-sdk/sidecar) |
| Serverless | Your function must flush events before each invocation exits | [Serverless](/integrations/usage/instrumentation-sdk/serverless) |
| CLI / Daemon | You want to pipe events from scripts, jobs, or shell pipelines | [CLI / Daemon](/integrations/usage/instrumentation-sdk/cli) |

## How it works

1. Generate an SDK API key from **Settings → API Keys → SDK Keys** in your Doow workspace (keys start with `dk_`)
2. Install the SDK or configure your delivery method (see sub-sections below)
3. Emit `track()` calls from your application code as usage occurs
4. Doow receives the events, deduplicates them, and attributes them to the matching license in your workspace

## Integration methods

Choose the method that fits your stack:

| Method | Best for |
|---|---|
| [Node.js SDK](/integrations/usage/instrumentation-sdk/nodejs) | Node.js applications — direct code integration with automatic batching and retry |
| [Sidecar](/integrations/usage/instrumentation-sdk/sidecar) | Non-Node.js applications — run a companion container that receives events over TCP or stdin |
| [Serverless](/integrations/usage/instrumentation-sdk/serverless) | AWS Lambda, Vercel, and Azure Functions — managed flush wrappers for short-lived runtimes |
| [CLI / Daemon](/integrations/usage/instrumentation-sdk/cli) | Scripted pipelines and shell integrations — pipe newline-delimited JSON events from any process |

## What Doow receives

Each event you emit includes:
- `metric` — the metric name (for example, `api_calls`, `tokens`, `storage_gb`)
- `quantity` — the numeric usage value
- `license_id` — which license in your Doow workspace to attribute it to
- Optional: `attribution` metadata for team, model, environment, or other custom dimensions

Doow does not receive any payload content from your application — only the structured metric events you explicitly emit.

## Confirm events arrived

After sending a test event, open the integration detail page from
**Company Settings → Integrations**. A healthy SDK integration should show the
event with the expected metric, quantity, license, and attribution fields.

If no event appears, check the `dk_` key, event shape, `license_id`, and flush
behavior before adding more instrumentation.

## Authentication

All SDK events are authenticated with your `dk_` API key. Keys can be created, rotated, and revoked from **Settings → API Keys → SDK Keys** without disconnecting your integration.
