Doow works best when it can combine usage data, organization data, and
financial data. You do not need to connect everything on day one. Start with
the source that matches the outcome you need first, then add the other data
sources as your review process matures.

## The three data layers

| Data layer | What it answers | Examples |
|---|---|---|
| Usage data | What is being consumed? | Browser activity, desktop app activity, tokens, API calls, model usage, events, requests, quota consumption |
| Organization data | Who uses it and where do they belong? | Users, groups, app assignments, departments, managers, employment status |
| Financial data | What did the company pay? | Expenses, vendors, account categories, transaction dates |

## Why Doow combines them

Usage alone can tell you that a tool is active, but it does not always explain
who owns the usage or whether the spend is justified. Identity and HRIS data
add people, ownership, department, and employment context. Accounting data adds
the finance-confirmed spend record.

When these layers work together, Doow can help you answer questions like:

- Which team is driving the most usage?
- Which active licenses are tied to departed employees?
- Which vendor costs are increasing faster than usage?
- Which tools need review before renewal?

## Minimum useful setup

For most teams, connect:

1. One usage source for the product, AI provider, app, or platform you want to
   inspect first. That can be Doow Agent, a provider API, SDK events,
   observability metrics, or OTLP.
2. One identity provider so Doow can match usage to people.

That gives Doow enough context to show what is being used and who is connected
to it. Add HRIS when you need departments and employment status. Add accounting
when you need finance-confirmed spend.
