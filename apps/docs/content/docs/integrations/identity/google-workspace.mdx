Connect your Google Workspace account to sync your organization's user directory and application activity into Doow.

## What you need

- A Google Workspace account with **Super Admin** or **Groups Admin** privileges
- Admin Console access to your Workspace domain

You do not need to create API credentials manually. Doow uses a standard OAuth flow — you simply authorize access through Google's consent screen.

## What Doow reads

### Users
- Full name (given name + family name)
- Primary work email address
- Account status (active or suspended)
- Organizational unit / department

### App activity
- Last sign-in time per user, per application, from Google's login audit logs
- Which Workspace-managed apps each user has access to

### Domain
- Your Workspace domain (for example, `yourcompany.com`) — used to scope all data to your organization

## What Doow does not read

- Passwords or authentication secrets
- Email content, calendar events, or Drive files
- Personal (non-Workspace) Google accounts
- Any data outside directory and audit logs

## How to connect

1. Go to **Company Settings → Integrations** in your Doow workspace
2. Find **Google Workspace** and click **Connect**
3. A popup window opens — Doow fetches the Google OAuth consent URL
4. The popup navigates to Google's OAuth consent screen
5. Sign in with a Workspace account that has Super Admin or Groups Admin privileges
6. Review the requested permissions and click **Allow**
7. The popup closes automatically and Doow completes the connection
8. A success notification confirms the integration is connected

> If your browser blocks the popup, you will be redirected to Google's consent screen directly. After approving, you are redirected back to the Integrations page and the connection completes automatically.

The initial sync may take a few minutes depending on the size of your directory. After that, Doow refreshes user and activity data on a scheduled basis.

## Permissions requested

Doow requests the following Google OAuth scopes:

| Scope | Purpose |
|---|---|
| `admin.directory.user.readonly` | Read user profiles and account status |
| `admin.reports.audit.readonly` | Read application login activity |

These are read-only scopes. Doow cannot create, modify, or delete any data in your Google Workspace.

## Confirm the sync worked

After connecting, open the integration detail page from **Company Settings → Integrations**. A healthy identity sync should show a connected state, a recent sync timestamp, imported users, and app assignment or activity records where the provider makes them available.

If the connection is successful but no users appear, verify that you connected the right tenant, granted admin consent, and allowed the required read scopes.

## Troubleshooting

Use these checks when the connection succeeds but data is missing or incomplete:

- Confirm you connected the intended tenant or domain.
- Confirm the admin consent screen granted the requested read permissions.
- Confirm the provider account can read users, groups, and app assignments.
- Check the integration event log for permission or sync errors.
- Trigger a manual sync when the integration supports it.

### Google Workspace-specific checks

| Symptom | Likely cause | Next action |
|---|---|---|
| OAuth completes but no users appear | The admin account cannot read directory users for the domain | Reconnect with a Google Workspace admin that can read the directory |
| App activity is missing | The granted consent does not include the needed activity scope | Review the consent screen and reconnect if scopes were denied |
| Wrong domain appears | The OAuth flow used the wrong Google account | Disconnect and reconnect with the intended Workspace admin account |

## Disconnecting

To disconnect, go to **Company Settings → Integrations**, find Google Workspace, and click **Disconnect**. This immediately stops all syncs and removes stored credentials. You can also revoke access directly from your [Google Account permissions page](https://myaccount.google.com/permissions).
