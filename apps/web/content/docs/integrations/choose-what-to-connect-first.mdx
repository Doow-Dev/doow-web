Use this page to choose the source family that matches the outcome you need.
The right connection is the one that gives Doow the missing data for the review
you are trying to run.

<Callout title="If you are not sure">
  Many teams begin with one usage source and one identity provider because that
  shows what is being used and who is connected to it. Use that as a baseline,
  not a required order.
</Callout>

## Choose by outcome

<Cards>
  <Card title="See usage" href="/docs/integrations/usage/choose-source">
    Use this when you need browser activity, desktop app activity, models,
    projects, services, events, or metrics.
  </Card>
  <Card title="Match usage to users" href="/docs/integrations/identity/google-workspace">
    Use this when you need users, groups, app access, and assignments from an

  </Card>
  <Card title="Attribute to departments" href="/docs/integrations/hris">
    Use this when departments, managers, job titles, or employment status

## Common paths

### Usage visibility

Use this path when the missing data is activity, consumption, events, requests,
tokens, services, or metrics.

<Steps>

1. Open [Choose a usage data source](/integrations/usage/choose-source).
2. Pick Doow Agent, direct provider APIs, cloud platforms, instrumentation,
   observability, or OTLP.
3. Connect the provider or method.
4. Confirm usage data appears in Doow.

</Steps>

### Ownership and user context

Use this path when the missing data is the person, group, assignment, team, or
employment context behind the usage.

<Steps>

1. Connect [Google Workspace](/integrations/identity/google-workspace),
   [Microsoft 365](/integrations/identity/microsoft-365), or another identity
   provider.
2. Confirm users and app assignments sync.
3. Add HRIS if you need departments, managers, and employment status.

</Steps>

### Spend reconciliation

Use this path when the missing data is the finance record behind software or
SaaS spend.

<Steps>

1. Connect [Accounting integrations](/integrations/accounting).
2. Select the expense categories that contain software and SaaS spend.
3. Confirm expenses appear in Doow.
4. Add usage and identity data so Doow can explain whether spend matches use.

</Steps>
